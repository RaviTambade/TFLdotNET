# ASP.NET Core Application Basics

## 1пёЏвѓЈ ASP.NET Framework vs ASP.NET Core

* **ASP.NET Framework**

  * Works mainly on Windows
  * Depends on IIS
  * Older project structure

* **ASP.NET Core**

  * Open source
  * Cross-platform (Windows, Linux, macOS)
  * Can run without IIS
  * Uses a **different folder structure**

рџ‘‰ **ASP.NET Core applications are lightweight and modern**

## 2пёЏвѓЈ Program.cs вЂ“ Entry Point

* `Program.cs` is the **starting point** of the application.
* Execution always begins from here.
* Similar to `main()` function in C/C++.

**Purpose of Program.cs**

* Configure the application
* Start the web server
* Listen for browser requests

## 3пёЏвѓЈ Creating WebApplication Builder

```csharp

```

* `CreateBuilder()` is an **inbuilt static method**
* It prepares:

  * Configuration
  * Services
  * Hosting environment

рџ‘‰ Builder = **Application setup object**

## 4пёЏвѓЈ Adding Services (MVC)


* Registers MVC features
* Enables:

  * Controllers
  * Views

рџ“Њ If a service is not added here, it **cannot be used** in the app.

## 5пёЏвѓЈ Building the Application

```csharp

```

* Converts setup into a **ready application**
* After this step, configuration is fixed

## 6пёЏвѓЈ Middleware Pipeline

* Middleware decides **how requests are processed**

Example:

```csharp

app.UseAuthorization();
app.UseSession();
```

* Browser request passes through middleware **one by one**
* Each middleware performs a specific task

рџ‘‰ Request в†’ Middleware в†’ Controller в†’ Response

## 7пёЏвѓЈ Static Files & wwwroot

* `wwwroot` folder contains:

  * CSS
  * JavaScript
  * Images

```csharp

```

* Static files are served **directly**
* Controllers are not involved


## 8пёЏвѓЈ Routing in MVC

```csharp

    name: "default",
    pattern: "{controller=Home}/{action=Index}/{id?}");
```

* Connects URL with controller and action
* Default URL:

  ```
  / в†’ HomeController в†’ Index()
  ```

## 9пёЏвѓЈ Starting the Application

```csharp

```

* Starts the web server
* Keeps application running
* Listens for incoming requests

рџ“Њ Without `app.Run()`, application will not start.

## рџ”џ Kestrel Web Server

рџ‘‰ Used automatically when app runs

## 1пёЏвѓЈ1пёЏвѓЈ Hosting & Port Number

  * Example: `http://localhost:50186`

рџ‘‰ Open browser + port number to access app

## 1пёЏвѓЈ2пёЏвѓЈ Controllers

## 1пёЏвѓЈ3пёЏвѓЈ Action Methods

* Methods inside controller

## 1пёЏвѓЈ4пёЏвѓЈ Views (Razor Pages)

## 1пёЏвѓЈ5пёЏвѓЈ Build Output
