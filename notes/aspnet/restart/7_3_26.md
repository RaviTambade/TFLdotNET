# ASP.NET Web API

## Difference between ASP.NET (.NET Framework) and ASP.NET Core

### 1. Platform Support

| Feature      | ASP.NET (.NET Framework)            | ASP.NET Core                |
| ------------ | ----------------------------------- | --------------------------- |
| Platform     | Windows Only                        | Cross Platform              |
| Supported OS | Windows Server                      | Windows, Linux, macOS       |
| Hosting      | IIS (Internet Information Services) | Kestrel, IIS, Nginx, Apache |

**Explanation**

* **ASP.NET (.NET Framework)** applications run **only on Windows**.

* They are usually hosted on **Windows Server using IIS**.

* **ASP.NET Core** applications are **cross-platform**.

* They can run on:

  * Windows
  * Linux
  * macOS

Example hosting servers:

* **Kestrel (default server)**
* IIS
* Nginx
* Apache

---

# ASP.NET Core Architecture (High Level)

```
Client (Browser / Mobile / API Consumer)
            │
            ▼
      Proxy Server
 (IIS / Nginx / Apache)
            │
            ▼
        Kestrel Server
            │
            ▼
      ASP.NET Core Middleware
            │
            ▼
        Controllers
            │
            ▼
       Business Services
            │
            ▼
         Database
```

### Request Flow

1. Client sends request.
2. Request reaches **Proxy Server**.
3. Proxy forwards request to **Kestrel Server**.
4. Request passes through **Middleware pipeline**.
5. Middleware calls **Controller**.
6. Controller uses **Business Services**.
7. Services interact with **Database**.
8. Response returned to client.

# Dependency Injection (DI)

### What is Dependency?

A component required by another component to work.

Example:

* Controller needs **Business Service**
* Service needs **Repository**
* Repository needs **Database**

---

### Banking Example

```
Bank Controller
      │
      ▼
Banking Service (Business Logic)
      │
      ▼
Database Access
```

Without **business logic**, the controller cannot perform useful work.

So,

* Controller **depends on** Service.
* Service is **injected into Controller**.

---

# What is Dependency Injection?

Dependency Injection is a **design pattern** used to create **loosely coupled applications**.

Instead of creating objects manually inside classes, the **framework provides the required objects automatically**.

Example:

```
Controller → Service → Repository
```

The framework **injects Service into Controller**.

Benefits:

* Loose coupling
* Easier testing
* Better maintainability
* Reusable code

---

# Dependency Injection in ASP.NET

| Framework                | Dependency Injection |
| ------------------------ | -------------------- |
| ASP.NET (.NET Framework) | Not built-in         |
| ASP.NET Core             | Built-in by default  |

ASP.NET Core has **inbuilt Dependency Injection container**.

---

# Relationship with SOLID Principles

Dependency Injection supports **SOLID design principles**.

### SOLID

**S — Single Responsibility Principle**
A class should have only **one responsibility**.

**O — Open/Closed Principle**
Open for **extension**, closed for **modification**.

**L — Liskov Substitution Principle**

**I — Interface Segregation Principle**

**D — Dependency Inversion Principle**

Dependency Injection mainly supports **Dependency Inversion Principle**.

---

# SQL Injection (Different Concept)

Do not confuse:

* **Dependency Injection**
* **SQL Injection**

They are **different concepts**.

### SQL Injection

A **security vulnerability** where hackers manipulate SQL queries.

Example problem:

```sql
SELECT * FROM Users
WHERE Username = 'admin' AND Password='123'
```

Attackers may modify query to access data.

Solution:

Use **parameterized queries**

Example:

```
command.Parameters.Add(...)
```

# 1. Shopping Cart Domain Concept

In an **E-commerce system**:

* Customer selects products
* Products are added to **Shopping Cart**
* Cart contains **Cart Items**

Relationship:

```
ShoppingCart
     │
     │ contains
     ▼
CartItem
```

This is called a **Containment (Has-A) Relationship** in Object Oriented Programming.

Example:

```
Cart has CartItems
```

---

# 2. Object-Oriented Relationships

There are mainly **two important relationships**:

### 1. Inheritance (IS-A)

Example:

```
Vehicle
   ↑
Car
```

Car **is a** Vehicle.

### 2. Containment (HAS-A)

Example:

```
Cart
  │
  └── CartItem
```

Cart **has** CartItems.

> Then Cart API manages selected products.
> Then Order API converts cart into order.
> Then Payment API handles payment.
> Finally Shipment API handles delivery.”
