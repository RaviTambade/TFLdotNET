# 5-Minute Whiteboard System Design Method

## Step 1 — Understand the Problem

First ask:

```
What are we building?
Who are the users?
What are the core features?
```

Example: **E-Commerce System**

Core features:

```
Browse products
Add to cart
Place order
Make payment
Track delivery
```


# Step 2 — Identify Main Components

Break the system into **major services**.

```text
                SYSTEM SERVICES

      +-----------+  +-----------+  +-----------+
      | Product   |  | Cart      |  | Order     |
      | Service   |  | Service   |  | Service   |
      +-----------+  +-----------+  +-----------+

      +-----------+  +-----------+
      | Payment   |  | Shipping  |
      | Service   |  | Service   |
      +-----------+  +-----------+
```

Each service becomes a **microservice**.

# Event Driven Communication

Large systems avoid tight coupling by using **event messaging**.

```text
                 +---------------------------+
                 |        MESSAGE BUS        |
                 |---------------------------|
                 | Kafka / RabbitMQ / SQS    |
                 +-------------+-------------+
                               |
      -------------------------------------------------------
        |                     |                     |
        v                     v                     v

   OrderCreated        PaymentCompleted       ShipmentStarted
      Event                 Event                 Event
```

Example flow:

```
Customer places order
        ↓
Order Service creates order
        ↓
Publish Event → OrderCreated
        ↓
Payment Service processes payment
        ↓
Publish Event → PaymentCompleted
        ↓
Shipping Service starts delivery
```

# The 10-Step Request Journey

 

```
1 User opens website
2 Request goes to CDN
3 Load balancer distributes traffic
4 API Gateway authenticates request
5 Microservice processes request
6 Cache checked for fast response
7 Database accessed if needed
8 Events published to message queue
9 Other services react asynchronously
10 Response returned to user
```
 

Students can remember this simple structure:

```
Users
  ↓
CDN
  ↓
Load Balancer
  ↓
API Gateway
  ↓
Microservices
  ↓
Cache + Database
  ↓
Message Queue
  ↓
Containers
  ↓
Cloud Infrastructure
```

# Understanding CLI, Node.js Project Structure, and .NET Runtime (Classroom Explanation)

## 1. What is CLI?

**CLI = Command Line Interface**

A **Command Line Interface** is a way to interact with a computer or development tools by **typing commands in a terminal instead of using graphical buttons**.

Examples of CLI tools used by developers:

* Node.js CLI
* .NET CLI
* Docker CLI
* Git CLI

Example commands:

```bash
npm init
dotnet new console
docker build
git clone
```

These commands **create projects, install libraries, and run applications**.


# 2. Node.js Project Creation using CLI

When we create a Node.js project, we normally use the command:

```bash
npm init
```

This command initializes a Node.js project.

### Files/Folders Created

```
project-folder
│
├── package.json
├── server.js / app.js
├── node_modules/
├── routes/
├── services/
├── repositories/
├── public/
│    └── index.html
```

### Important Components

**package.json**

Contains project information:

* project name
* version
* dependencies
* scripts

Example:

```json
{
 "name": "ecommerce-app",
 "version": "1.0.0",
 "dependencies": {
   "express": "^4.18.2"
 }
}
```

### node_modules

This folder contains **downloaded libraries**.

Example libraries:

* Express
* Axios
* Mongoose

Installed using:

```bash
npm install express
```

### Typical Backend Structure

```
src
│
├── controllers
├── services
├── repositories
├── routes
└── models
```

This structure is commonly used in **microservice architecture**.

# 3. Containerized Applications

Modern applications are deployed using **containers**.

Important files:

### Dockerfile

Defines how the application container should be built.

Example:

```dockerfile
FROM node:18
WORKDIR /app
COPY . .
RUN npm install
CMD ["node","server.js"]
```

### docker-compose.yml

Used to run **multiple containers together**.

Example:

* Web API container
* Database container
* Redis container

# 4. Runtime Architecture in Different Technologies

When we deploy applications, they run on **specific runtimes**.

### Java Runtime

Components:

```
JRE
 ├── JVM
 ├── Core Libraries
 └── JAR files
```

Explanation:

* **JVM (Java Virtual Machine)** → executes bytecode
* **JAR files** → compressed Java class files
* Frameworks:

  * Spring Boot
  * Hibernate
  * JPA


### .NET Runtime

Components:

```
CLR
 ├── .NET Core Libraries
 ├── DLL files
 └── Application Code
```

Explanation:

* **CLR (Common Language Runtime)** executes .NET programs
* .NET applications compile into **DLL files**

Example frameworks:

* ASP.NET Core
* MVC
* Web API


# 5. Types of Applications in .NET

Using CLI command:

```bash
dotnet new
```

You can create different types of applications.

### Console Application

```bash
dotnet new console
```

Used for:

* CLI tools
* learning programming
* backend utilities


### Web Applications

Examples:

* ASP.NET MVC
* ASP.NET Web API

Used for:

* websites
* REST APIs
* microservices


### Windows Applications

Example technologies:

* Windows Forms
* WPF

Used for **desktop applications**.


### Worker Services

Used for **background processing**

Examples:

* message queue consumers
* scheduled jobs
* data processing services

| .NET       | CLR                |
| Node.js    | V8 Engine          |
| Python     | Python Interpreter |
| Go         | Go Runtime         |
| C/C++      | Native Runtime     |
