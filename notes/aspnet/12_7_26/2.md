# Session 2

###  ASP.NET Core, React, AI Engineering, Package Management, CORS, Middleware, and Enterprise Development


## Session Objective

In this session, Ravi Tambade explained how modern enterprise software development has evolved from simply writing code to building AI-enabled, cloud-ready applications using ASP.NET Core, React, package managers, middleware, dependency injection, and modern software architecture.

The discussion also emphasized that developers should focus more on solving business problems and system design than writing repetitive code.


## 1. Package Management in Modern Development

Every modern programming platform has its own package manager to install reusable libraries.

Instead of writing every functionality from scratch, developers reuse trusted libraries.

### Package Managers

| Platform | Package Manager | Configuration File |
| -------- | --------------- | ------------------ |
| .NET     | NuGet           | .csproj            |
| Node.js  | npm             | package.json       |
| Java     | Maven           | pom.xml            |
| Python   | pip             | requirements.txt   |

### React Project

React stores project information inside

```
package.json
```

This file contains

* Project name
* Dependencies
* Scripts
* Version numbers
* Build configuration

Example

```json
{
  "name": "insurance-ui",
  "dependencies": {
      "react": "^19.0.0",
      "axios": "^1.9.0"
  }
}
```

## 16. Cross-Origin Resource Sharing (CORS)

React and ASP.NET Core usually run on different ports during development.

Example

```
React
http://localhost:5173
↓
ASP.NET Core

https://localhost:7001
```

Browsers block requests between different origins unless explicitly allowed. ASP.NET Core enables this using CORS.

Example

```csharp
builder.Services.AddCors();
```

```csharp
app.UseCors();
```

This allows trusted front-end applications to access the API.

## 19. Application Startup

Typical ASP.NET Core startup sequence

```
Program.cs
↓
Builder
↓
Register Services
↓
Build Application
↓
Configure Middleware
↓
Map Controllers
↓
Run Application
```

Example

```csharp
builder.Services.AddControllers();
var app = builder.Build();
app.MapControllers();
app.Run();
```

This minimal hosting model replaces the older Startup.cs pattern.

# 22. Dependency Injection

Business services are registered during startup.

Example

```csharp
builder.Services.AddScoped<IProductService, ProductService>();
```

Controllers receive these services through constructor injection, promoting loose coupling and easier testing.

## 24. Practical Development Advice

Ravi concluded with a pragmatic principle often followed in production systems:

> **"If something is working, don't touch it."**

This highlights the importance of avoiding unnecessary changes to stable code. Refactor or modify software only when there is a clear business need, bug fix, performance improvement, or maintainability benefit.


## Key Takeaways

* Use **package managers** (`npm`, `NuGet`, `Maven`, `pip`) to manage project dependencies.
* Build applications using **open-source libraries** rather than reinventing common functionality.
* Understand **version compatibility** between frameworks and packages.
* Integrate AI capabilities using **Microsoft.Extensions.AI** and **Semantic Kernel**.
* Use **Axios** in React for REST API communication.
* Develop a broad understanding of modern technologies while gaining deep expertise in one area.
* Learn enterprise practices such as **SOLID**, **DRY**, **KISS**, **Dependency Injection**, **Microservices**, **Docker**, **Azure**, and **CI/CD**.
* Recognize that AI assists development, but engineers remain responsible for architecture, validation, testing, and correcting hallucinations.
* Prefer **TSX** for large React applications due to its type safety.
* Configure **CORS** so React frontends can securely communicate with ASP.NET Core APIs.
* Understand the ASP.NET Core startup flow and middleware pipeline centered around **Program.cs**.
* Build software **incrementally**, delivering one complete, testable feature at a time.
