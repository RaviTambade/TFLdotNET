## 1. Dependency Injection (DI)

### What is Dependency Injection?

Dependency Injection is a design pattern in which objects receive their required dependencies from the framework instead of creating them manually.

Instead of:

```csharp
var service = new ProductService();
```

ASP.NET Core automatically creates and injects the object.



###  Registering Services

Services are registered inside `Program.cs`.

```csharp
builder.Services.AddScoped<IProductService, ProductService>();
```

Explanation:

* `IProductService` → Interface (contract)
* `ProductService` → Concrete implementation
* `AddScoped()` → One instance per HTTP request

###  Why Use Interfaces?

The controller depends on the interface rather than a concrete class.

```text
Controller
     │
     ▼
IProductService
     │
     ▼
ProductService
```

Benefits:

* Loose coupling
* Easy unit testing
* Easy replacement of implementations
* Better maintainability


###  Constructor Injection

ASP.NET Core injects dependencies automatically.

```csharp
public class ProductsController : ControllerBase
{
    private readonly IProductService _service;

    public ProductsController(IProductService service)
    {
        _service = service;
    }
}
```

The controller focuses on handling HTTP requests, while business logic resides in the service layer.

## 3. RabbitMQ and Asynchronous Communication

###  Why RabbitMQ?

Direct service-to-service communication creates tight coupling.

Instead:

```text
Claim Service
↓
RabbitMQ
↓
Notification Service
```

RabbitMQ acts as a message broker.


###  Publisher–Subscriber Model

```text
Publisher
↓
Queue
↓

Subscriber
```

Example:

* Premium Payment Service publishes a **PaymentCompleted** event.
* Notification Service subscribes and sends an email.
* Audit Service subscribes and records the transaction.
* Reporting Service subscribes and updates reports.

Each service works independently.


### Benefits

* Loose coupling
* Reliable messaging
* Offline processing
* Better scalability
* Fault tolerance


## 4. Event-Driven Workflow

Example:

```text
Customer uploads documents
↓
Document Verification
↓
Inspection
↓
Approval
↓
Policy Activation
↓
Notification
```

Each stage publishes an event. Other services react to those events without direct dependencies.


## 5. Communication Types

### Synchronous

```text
Client
↓
Service
↓
Wait
↓
Response
```

The client blocks until a response is received.

### Asynchronous

```text
Client
↓
Queue
↓
Continue Working
↓
Service Processes Later
↓
Notification
```

The client does not wait, improving responsiveness and scalability.

## 6. API Testing

During development, APIs should be tested independently of the frontend. Common tools:

* Swagger UI
* Postman

Typical endpoints:

```http
GET    /api/products
GET    /api/products/{id}
POST   /api/products
PUT    /api/products/{id}
DELETE /api/products/{id}
```

Swagger generates interactive API documentation automatically.


## 7. ASP.NET Core and Microsoft.Extensions.AI

The AI libraries follow the same Dependency Injection model as other ASP.NET Core services.

Example:

```csharp
builder.Services.AddChatClient(...);
```

This consistency means developers can integrate AI into existing applications without changing architectural principles.


## 8. AI Should Enhance Existing Systems

A key message from Ravi was:

> Do not rebuild working business systems.

Instead:

* Keep existing databases.
* Reuse existing APIs.
* Add AI capabilities on top.

Architecture:

```text
Existing ERP
↓
Existing APIs
↓
AI Layer
↓
Chatbot / Assistant
↓
Users
```

AI complements enterprise software rather than replacing it.


## 9. The AI Technology Stack

Ravi revisited the AI stack discussed earlier.

```text
Applications
      ↑
Large Language Models
      ↑
Data Centers
      ↑
Data Infrastructure
      ↑
Semiconductor Chips
      ↑
Electricity
```

### Layers Explained

1. **Electricity** – powers the infrastructure.
2. **Semiconductor Chips** – CPUs, GPUs, TPUs perform computation.
3. **Data Infrastructure** – servers, storage, networking, vector databases.
4. **Data Centers** – cloud infrastructure hosting AI workloads.
5. **Large Language Models (LLMs)** – Gemini, GPT, Claude, etc.
6. **Applications** – chatbots, copilots, enterprise assistants.

Understanding the full stack helps developers appreciate how AI systems are built and deployed.

## 14. JWT Authentication

React applications authenticate using JSON Web Tokens (JWT). Workflow:

```text
User Login
↓
Access Token
↓
API Requests
↓
Token Expires
↓
Refresh Token
↓
New Access Token
```

This avoids forcing users to log in repeatedly.


## 15. React Interceptors

Ravi discussed using Axios interceptors to automate token refresh.

Typical flow:

```text
React Request
↓
Axios Interceptor
↓
Access Token Attached
↓
API Call
↓
401 Unauthorized?
↓
Call Refresh Token API
↓
Receive New Token
↓
Retry Original Request
```

This keeps authentication transparent to the user.



## 16. Prompt Engineering

One of the most important AI skills is asking effective questions.Good prompts produce better results. Instead of asking:

> "Write code"

Ask:

> "Create an Axios response interceptor in React that automatically refreshes an expired JWT using a refresh token API and retries the original request."

Well-defined prompts improve the quality of AI-generated solutions.

## Key Takeaways

* Use **Dependency Injection** to decouple controllers from service implementations.
* Depend on **interfaces** rather than concrete classes to improve flexibility and testability.
* As applications grow, shift from direct service calls to **event-driven microservices** using **RabbitMQ** or similar message brokers.
* Test REST APIs independently with **Swagger** and **Postman** before integrating the frontend.
* Add AI capabilities **on top of existing enterprise systems** instead of rebuilding mature databases and APIs.
* Understand the complete AI technology stack, from **electricity and semiconductor chips** to **LLMs and end-user applications**.
* Continue mastering **ASP.NET Core**, **React**, and **microservices** while gradually learning **AI**, **Machine Learning**, **Deep Learning**, **LLMs**, and **RAG**.
* Remember that **AI enhances software engineering but does not replace engineering fundamentals**.
* Implement secure authentication using **JWT access tokens** and **refresh tokens**, and automate renewal with **Axios interceptors** in React.
* Develop strong **prompt engineering** skills to work effectively with AI tools.
* Embrace lifelong learning with the mindset: **Learn → Unlearn → Relearn**.
