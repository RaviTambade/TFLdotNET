## 1. Domain Driven Development (DDD)

### What is Domain?

A **Domain** is the business area for which software is being built.

Examples

* Banking Domain
* Insurance Domain
* Hospital Domain
* E-Commerce Domain

In this course the domain is

> **Insurance Management System**


### What is Domain Driven Development?

Instead of designing software around database tables or UI screens,

we design software around

> **Business Concepts**

Software should represent the language spoken by business experts.

Example

Insurance experts talk about

* Customer
* Policy
* Premium
* Claim
* Agent
* Nominee
* Renewal

These become the core classes inside our software.

Example


## 11. Asynchronous Programming

Large operations should never block application. Example

Wrong

```
Customer
↓
Click Payment
↓
Application waits
↓
Database waits
↓
Everything blocked
```

Blocking causes
* Slow application
* Timeout
* Poor scalability

Correct approach

```
Customer
↓
Send Request
↓
Task Starts
↓
Customer Thread Free
↓
Database Working
↓
Result Returns Later
```

This is called Non-Blocking Programming


###  Restaurant Analogy

Customer orders
↓
Waiter sends order
↓
Chef cooks
↓
Customer keeps talking
↓

Food arrives later
Nobody waits doing nothing.
Same happens in async programming.


### C# Example

```csharp
public async Task<PaymentDTO> MakePayment()
{
    return await repository.MakePaymentAsync();
}
```

The thread is released while database is working.


## 12. Thread Pool

Creating threads manually is expensive.Instead .NET maintains Thread Pool

```
Application Process

-------------------------

Primary Thread

Thread Pool
    Thread 1
    Thread 2
    Thread 3
    Thread 4
```

Whenever Task starts Available thread executes it. Developer only creates Tasks NOT Threads.



## 13. Why Task instead of Thread?

Thread
* Heavy
* Expensive
* Difficult to manage

Task
* Lightweight
* Uses Thread Pool
* Recommended by Microsoft

Always prefer
```
Task
async
await
```

## Key Takeaways

* Design software around the **business domain**, not around database tables.
* Build software **one user story at a time** using Agile sprints.
* Use **DDD** to model business entities such as Customer, Policy, Premium, and Claim.
* Follow **TDD** to improve code quality by writing tests before implementation.
* Separate responsibilities using **Controller → Service → Repository → Database** architecture.
* Use **DTOs** to transfer data securely between client and server.
* Prefer **async/await** and **Task** for scalable, non-blocking applications.
* Let the **.NET Thread Pool** manage threads instead of creating them manually.
* Understand the ASP.NET Core request pipeline: **Kestrel → Middleware → Routing → Controller → Service → Repository → Database → Response**.
* Build responsive frontends with **React** as a Single Page Application.
* Use package managers and CLI tools (`dotnet`, `npm`, `mvn`, `pip`) to scaffold projects and manage dependencies efficiently.
* Develop enterprise software **incrementally**, delivering working features sprint by sprint rather than attempting the entire application at once.
