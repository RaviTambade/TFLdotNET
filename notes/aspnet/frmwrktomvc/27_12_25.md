

# 🧠 First, Reset the Mindset (WebForms → MVC)

### How you worked earlier (WebForms)

```text
UI (ASPX)
 └── Code Behind (ASPX.CS)
      ├── Page_Load
      ├── Button_Click
      └── ViewState + PostBack
```

### How ASP.NET Core MVC works

```text
Request
  ↓
Controller  → Business Logic / Service
  ↓
View (Razor .cshtml)
```

🔴 **No Page_Load**
🔴 **No ViewState**
🔴 **No server controls (TextBox, GridView)**
✅ Clean **HTTP request–response model**

Think like this:

> “A web app is just HTTP requests handled by methods.”

# 1️⃣ What is ASP.NET Core MVC?

**MVC = Model – View – Controller**

| Part           | Responsibility        |
| -------------- | --------------------- |
| **Model**      | Data + business rules |
| **View**       | UI (HTML + Razor)     |
| **Controller** | Handles HTTP requests |

👉 MVC enforces **Separation of Concerns**
👉 WebForms mixed everything together

# 4️⃣ Controller – The Heart of MVC

### HomeController.cs

```csharp
public class HomeController : Controller
{
    public IActionResult Index()
    {
        return View();
    }

    public IActionResult Privacy()
    {
        return View();
    }
}
```

### What’s happening?

* URL: `/Home/Index`
* Method executes
* Returns a **View**

🧠 Think:

> “Each public method = one HTTP endpoint”

# 5️⃣ View – Razor (.cshtml)

### Views/Home/Index.cshtml

```html
@{
    ViewData["Title"] = "Home Page";
}

<h1>Welcome to ASP.NET Core MVC</h1>
<p>Hello from Razor View</p>
```

### Razor Basics

| Syntax     | Meaning      |
| ---------- | ------------ |
| `@{ }`     | C# block     |
| `@Model`   | Access model |
| `@foreach` | Loop         |
| `@if`      | Condition    |

⚠️ No server controls
Everything is **HTML-first**

# 6️⃣ Model – Passing Data to View

### Model

```csharp
public class Product
{
    public int Id { get; set; }
    public string Name { get; set; }
    public decimal Price { get; set; }
}
```

### Controller

```csharp
public IActionResult Index()
{
    var product = new Product
    {
        Id = 1,
        Name = "Laptop",
        Price = 50000
    };

    return View(product);
}
```

### View

```html
@model Product

<h2>@Model.Name</h2>
<p>Price: ₹@Model.Price</p>
```

🧠 This replaces:

* DataSet
* DataTable
* GridView binding

---

# 7️⃣ Layout (_Layout.cshtml) – Master Page Replacement

```html
<!DOCTYPE html>
<html>
<head>
    <title>@ViewData["Title"]</title>
</head>
<body>
    <header>My App</header>

    <main>
        @RenderBody()
    </main>

    <footer>© 2025</footer>
</body>
</html>
```

### WebForms Mapping

```
MasterPage.master → _Layout.cshtml
ContentPlaceHolder → @RenderBody()
```

# 8️⃣ Forms & Model Binding (Very Important Shift)

### Form

```html
<form asp-action="Create" method="post">
    <input asp-for="Name" />
    <input asp-for="Price" />
    <button type="submit">Save</button>
</form>
```

### Controller

```csharp
[HttpPost]
public IActionResult Create(Product product)
{
    // Model binding happens automatically
    return RedirectToAction("Index");
}
```

🧠 No Request.Form
🧠 No FindControl
🧠 Clean binding

---

# 9️⃣ Dependency Injection (Built-in in Core)

### Program.cs

```csharp
builder.Services.AddScoped<IProductService, ProductService>();
```

### Controller

```csharp
public class ProductController : Controller
{
    private readonly IProductService _service;

    public ProductController(IProductService service)
    {
        _service = service;
    }
}
```

👉 This replaces:

* Static helpers
* Global variables
* Tight coupling

> *Visual Studio menu options → converted into commands*

---

## 1️⃣ Check .NET SDK Installed

```bash
dotnet --version
```

✔ Confirms .NET SDK is available

---

## 2️⃣ List Available Project Templates

```bash
dotnet new list
```

✔ Shows templates like `mvc`, `webapi`, `console`, etc.

---

## 3️⃣ Create ASP.NET Core MVC Application

```bash
dotnet new mvc -n MyMvcApp
```

📌 What happens:

* Creates MVC project
* Adds Controllers, Views, Models
* Configures Program.cs

📁 Folder created:

```
MyMvcApp/
```

---

## 4️⃣ Move into Project Folder

```bash
cd MyMvcApp
```

---

## 5️⃣ Restore NuGet Packages

```bash
dotnet restore
```

✔ Downloads required dependencies
📌 Usually auto-done during build

---

## 6️⃣ Build the Application

```bash
dotnet build
```

✔ Compiles code
✔ Catches compile-time errors
📌 Like **Build Solution** in Visual Studio

---

## 7️⃣ Run the Application

```bash
dotnet run
```

✔ Starts Kestrel web server
✔ Shows URLs like:

```
http://localhost:5000
https://localhost:5001
```

Open in browser 🚀

---

## 8️⃣ Run with Hot Reload (Very Useful)

```bash
dotnet watch run
```

✔ Automatically reloads app on code changes
✔ Similar to **Edit & Continue**

---

## 9️⃣ Clean Build Output

```bash
dotnet clean
```

✔ Removes `bin` and `obj` folders

---

## 🔁 Full Flow (Most Common)

```bash
dotnet new mvc -n MyMvcApp
cd MyMvcApp
dotnet run
```

That’s it — application up and running 👌
