And your final classroom point is very important:

When students see C++ code, they should identify it not only by extension but also by language constructs.

For example:

## How to Identify C++ Code

### File Extension

* `.cpp`
* `.cc`
* `.cxx`

### C++ Features

* `cout`
* `cin`
* `<<`
* `>>`
* classes
* constructors
* templates
* namespaces
* references

Example:

```cpp
#include <iostream>
using namespace std;

int main()
{
    cout << "Hello";
    return 0;
}
```

The presence of:

```cpp
cout << 
cin >>
```

indicates:

* insertion operator `<<`
* extraction operator `>>`

which are part of C++ stream handling.

Whereas C code typically uses:

```c
printf();
scanf();
```

This kind of comparative teaching helps students understand:

* evolution of languages
* abstraction layers
* why languages were created
* how developer productivity increased
* what trade-offs happened

And that is exactly how strong mentors develop engineering maturity in students.
